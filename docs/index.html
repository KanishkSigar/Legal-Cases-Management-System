What it does

📁

Case management

Case numbers, types, a full status workflow (Open → Won/Lost), assigned lawyer, linked client, and a notes timeline on every case.

📅

Appointments

Schedule meetings between a lawyer and a client, optionally tied to a case, with one-click complete / cancel.

👥

Lawyers & clients

Full profile management — specialisation, bar number, company, contact — each backed by a real login account.

🔐

Role-based access

Separate admin & client/lawyer login portals. Admins see all; lawyers see only their cases; clients see only their own.

🪪

Self-service sign-up

Clients register in seconds with just an email, sign in with that email, and reset a forgotten password by mail.

🔎

Search & sort

Find cases by number, title or client and appointments by person; sort and paginate large lists.

✅

Tested & CI

A Django test suite covering auth, role-scoping and reset runs automatically on every push via GitHub Actions.

Built around a relational core

The whole practice is modelled by five well-normalised tables. Foreign keys keep every record consistent; a for_user() manager scopes rows by role in SQL.

 ┌──────────────┐      ┌───────────────┐      ┌──────────────┐
 │     User     │ 1  1 │ LawyerProfile │ 1  * │     Case     │
 │  role, phone ├──────┤  specialism   ├──────┤ case_number  │
 │  (auth)      │      │  bar_number   │      │ status, type │
 └──────┬───────┘      └───────────────┘      └──────┬───────┘
        │ 1          1 ┌───────────────┐             │ 1
        └──────────────┤ ClientProfile ├── * ────────┘
                       │ company, addr │             │ *
                       └───────┬───────┘      ┌───────┴───────┐
                               │ *            │   CaseNote    │
                       ┌───────┴───────┐      │  author, text │
                       │  Appointment  │      └───────────────┘
                       │ scheduled_for │
                       │ status, case  │
                       └───────────────┘
  • Custom User + role — one auth table extended with a role column drives every permission.
  • Profiles via 1-to-1 — domain data kept out of auth.
  • Protected foreign keys — on_delete=PROTECT means no orphaned legal records.
  • Query-level scoping — a client literally cannot load another client's case.
