Security & data

Access control that lives in the database

Five normalised tables, joined by foreign keys, with a for_user() manager that scopes every query by role.

 ┌──────────────┐      ┌───────────────┐      ┌──────────────┐
 │     User     │ 1  1 │ LawyerProfile │ 1  * │     Case     │
